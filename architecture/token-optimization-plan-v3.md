## 核心发现

1. **模型已最便宜**（DeepSeek免费）→ 成本优化空间有限
2. **真正浪费来自：**
   - Cron session堆积（Beta 297个，5MB+）
   - 记忆系统配置不足（Flush阈值太低、搜索未启用）
   - 心跳上下文加载（11个Agent×30分钟×大量输入）
3. **优化重点：** 系统级配置 + 输入端压缩 + session管理

## 风险与应对

| 风险 | 应对 |
|------|------|
| 心跳频率降低导致消息延迟 | 事件驱动确保即时消息，核心Agent保持高频率 |
| 上下文压缩丢失信息 | 保留核心指令，关键信息永久标记 |
| Cron清理误删活跃session | 仅清理闲置>24小时的session |
| 配置变更导致系统异常 | 备份配置，<5分钟回滚 |
