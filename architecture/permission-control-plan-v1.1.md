## 1. 问题定义

### 1.1 当前安全风险
1. **网关重启权限过大** — 所有Agent都可执行 `openclaw gateway restart`，可能造成服务中断
2. **命令执行无限制** — Agent可执行任意系统命令，存在安全威胁
3. **权限提升漏洞** — 通过命令组合可能绕过现有权限控制

### 1.2 梦想家要求
- **重启网关为最高级别权限**，仅梦想家可以
- 建立四级权限体系
- 实施Exec白名单+黑名单配置
- 插件白名单管理
- Hooks安全加固
- 各Agent安全规则明确化
- OpenClaw per-agent命令限制机制

## 2. 四级权限体系设计

### 2.1 权限层级定义

```
Level 0 | 梦想家 🧠 (最高权限)
  - 网关重启、停止、配置变更（独占）
  - Agent创建、删除、权限管理
  - 系统级安全操作

Level 1 | 核心Agent 🛡️ (高权限)
  - 安全监控、审计、威胁检测
  - 权限管理辅助操作
  - 紧急响应操作
  - 审计级Exec权限（可执行监控命令，不可修改配置）

Level 2 | 功能Agent 🔧 (标准权限)
  - 常规技能执行
  - 文件读写操作
  - API调用
  - Git操作（按职责需要）

Level 3 | 受限Agent 📚 (最小权限)
  - 只读操作
  - 知识库查询
  - 信息展示
```

## 3. 安全威胁分析

### 3.1 威胁识别

**高危威胁：**
1. **网关服务中断** — Agent恶意重启网关（评分9 🔴）
2. **数据泄露** — Agent访问敏感文件（评分8 🔴）
3. **权限提升** — 通过漏洞获取更高权限（评分8 🔴）
4. **数据完整性破坏** — 配置文件篡改（openclaw.json、.env）（评分8 🔴）【v1.1新增】
5. **凭证泄露** — API token泄露+轮换缺失（评分8 🔴）【v1.1新增】
6. **供应链攻击** — 恶意技能安装（评分7 🔴）

**中危威胁：**
7. **权限滥用** — 合法权限被恶意使用，如Level 1创建恶意Agent（评分7 🟡）【v1.1新增】
8. **命令注入** — 通过参数传递恶意命令（评分6 🟡）
9. **级联故障** — 一个Agent被攻破后通过通信影响其他Agent（评分6 🟡）【v1.1新增】
10. **横向移动** — 攻击其他Agent或系统组件（评分5 🟡）
11. **配置漂移** — 权限配置被无意修改，安全边界模糊（评分5 🟡）【v1.1新增】

**低危威胁：**
12. **审计日志篡改** — 高权限Agent修改日志掩盖操作（评分4 🟡）【v1.1新增】
13. **拒绝服务** — 资源消耗攻击（评分3 🟢）
14. **信息收集** — 收集系统配置信息（评分4 🟢）
15. **日志污染** — 干扰安全监控（评分2 🟢）

## 4. Exec白名单配置方案

### 4.1 OpenClaw Exec安全机制
- **安全模式**: deny / allowlist / full
- **批准机制**: 高危命令需要用户批准
- **环境控制**: 可限制工作目录、环境变量

### 4.2 全局禁止命令（任何Agent不可执行）

```yaml
global_blacklist:
  - "rm -rf /"
  - "rm -rf /*"
  - "chmod 777"
  - "kill -9 1"
  - "curl | sh"
  - "wget | sh"
  - "> /dev/sda"
  - "dd if=/dev/zero"
  - "mkfs"
  - ":(){ :|:& };:"  # fork bomb
```

### 4.3 各Agent Exec白名单（v1.1 修订版）

**Level 1 Agent（核心安全Agent — 阿尔法/贝塔/艾普西隆）：**
```yaml
exec:
  security_mode: allowlist
  allowed:
    openclaw: [gateway status, cron list, agents list, sessions list]
    system_monitoring: [ps, top, df, free, uptime, iostat]
    logs: [journalctl, tail, grep, awk, wc]
    network: [netstat, ss, ping, curl -s]
    files_read: [cat, head, tail, find, ls]
    # 贝塔额外：可执行健康检查脚本
    # 阿尔法额外：可执行Agent管理命令
  forbidden:
    - "openclaw gateway restart"  # 仅Level 0
    - "openclaw gateway stop"
    - "rm -rf"
    - "chmod 777"
    - "kill -9"
```

**Level 2 Agent（功能Agent — 伽马/德尔塔/约塔/泽塔/艾塔/西塔/拉姆达/卡帕）：**
```yaml
exec:
  security_mode: allowlist
  allowed:
    files: [ls, cat, find, grep, head, tail, cp, mv, mkdir]
    git: [clone, pull, push, commit, status, log, diff, add]
    scripts: [node, python3]  # 注意：不含 bash 通配
    specific:  # 按Agent职责细分
      gamma: [npm, npx, clawhub]
      delta: [python3, pytest, ab, wrk]
      iota: [cat, grep, sort, awk]
      zeta: [crontab, systemctl, bash scripts/*/]  # 限定目录
      eta: [git, curl, tar, gzip]
      theta: [curl, cat, grep]
      lambda: [node, python3, curl, git]
      kappa: [convert, inkscape, node]
  forbidden:
    - "rm -rf"
    - "bash -c"  # 防止任意命令执行
    - "python3 -c"  # 防止任意代码执行
    - "chmod 777"
    - "kill -9"
    - "openclaw gateway restart"
```

**Level 3 Agent（受限只读Agent — 当前无Agent，预留）：**
```yaml
exec:
  security_mode: allowlist
  allowed:
    query: [grep, cat, head, tail]
    stats: [wc, sort, uniq]
  forbidden: [all write/execute operations]
```

## 5. 插件白名单管理

### 5.1 插件分类
- **系统插件**: gateway管理、权限控制
- **功能插件**: 技能执行、文件操作
- **监控插件**: 安全监控、日志分析
- **通信插件**: 外部API调用、消息发送

### 5.2 权限对应关系
```
Level 0: 所有插件
Level 1: 系统插件(部分) + 监控插件 + 通信插件
Level 2: 功能插件 + 通信插件(受限)
Level 3: 只读插件 + 查询插件
```

## 7. 各Agent安全规则

### 7.1 通用安全规则
1. **最小权限原则** — 每个Agent只获得完成任务所需的最小权限
2. **职责分离** — 安全、功能、监控职责明确分离
3. **操作审计** — 所有高危操作记录详细审计日志
4. **信任链验证** — 命令执行前验证调用链合法性

### 7.2 特别Agent安全规则

**艾普西隆 (安全官) 🛡️：**
- 唯一权限：可调用安全监控命令
- 限制：不能重启网关，只能监控状态
- 审计要求：所有安全操作双日志记录

**贝塔 (监察者) 🔵：**
- 权限：系统配置查看 + 审计级Exec（可执行监控脚本，不可修改配置）
- 限制：不能修改任何配置文件
- 监控：所有操作需同步报告给艾普西隆

**阿尔法 (CEO) 🦐：**
- 权限：Agent创建、权限分配
- 限制：新Agent默认Level 3，升级需梦想家批准
- 控制：权限分配需双重验证（至少一个Level 1确认）
