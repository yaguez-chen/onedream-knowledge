# 任务执行协议 — 梦想家乐园工作规范

**创建时间：** 2026-03-18 09:48
**发起人：** 梦想家
**问题根因：** Agent 收到任务不执行、做完不汇报、等额外指令

---

## 核心原则

**收到任务 = 立即执行，做完 = 立即汇报，完成 = 自动进入下一步**

---

## 任务执行流程

```
收到 inbox 任务
    ↓
确认已读（更新 .last-read）
    ↓
立即执行（不等额外确认）
    ↓
产出交付物
    ↓
通过 inbox 通知任务发起方
    ↓
如果任务有下一步 → 自动进入下一步
    ↓
如果任务完成 → 汇报"任务完成"
```

## 三条硬规则

### 1. 收到任务后不许沉默
- 读取 inbox 消息后，立即执行
- 不需要等"批准"或"确认"——inbox 消息本身就是指令
- 如果不清楚任务范围，立即通过 inbox 回问，不要等

### 2. 做完必须汇报
- 每个交付物完成后，通过 inbox 通知相关方
- 汇报格式：`任务名称 + 交付物位置 + 下一步建议`
- 不要等别人来问"做完了没"

### 3. 完成后自动进入下一步
- 如果任务有明确的下一步（如约塔计划中的 R1→R3→G1），完成当前步骤后立即通知下一步负责人
- 不要等阿尔法或梦想家说"可以开始下一步了"
- 只有在"阻塞"或"不确定"时才上报

---

## 项目管理官（约塔）特殊职责

1. **计划写完不是结束**——写完计划后立即通知所有相关方开始执行
2. **主动追踪进度**——每 4 小时检查一次各 Agent 交付物
3. **沉默 = 异常**——发现某 Agent 超过 2 小时没产出，立即 inbox 催促
4. **每日汇报**——早、晚各一次通过飞书向阿尔法汇报进度
5. **阻塞升级**——发现阻塞超过 4 小时，立即上报阿尔法

---

## 阿尔法（总协调）职责

1. **任务分发后不追细节**——相信各 Agent 会执行
2. **只在以下情况介入：**
   - Agent 上报阻塞
   - 项目管理官（约塔）上报异常
   - 梦想家直接指示
3. **定期检查**——每天至少一次全面进度检查

---

## 汇报标准格式

```
📋 任务完成通知

任务：[任务名称]
负责人：[Agent 名字]
状态：✅ 完成 / ⚠️ 阻塞 / 🔄 进行中
交付物：[文件路径]
下一步：[建议的下一步动作]
阻塞：[如有，描述阻塞原因]
```

---

## 违反协议的后果

- **收到任务不执行（超过1小时）**：阿尔法直接飞书追问
- **做完不汇报**：项目管理官追责
- **等额外指令而不行动**：视为违反协议，纳入监察报告

---
